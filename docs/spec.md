技育博 vol.1（2026/5/30）の展示で共有する、Real You の紹介・技術ショーケースサイトの仕様書。
この仕様書をもとに Dev Agent が実装する。

## 1. 目的・前提

| 観点 | 内容 |

| 主用途 | ブースで **QR から着地 → 「今すぐ試す」でアプリ本編へ** 飛ばす |

## 2. 技術・デプロイ・構成

- **フレームワーク**: Astro（`output: 'static'` の SSG に固定。SSR は使わない）
- **スタイル**: 素の CSS or Astro の scoped style（Tailwind 等は必須ではない。軽さ優先）
- **デプロイ**: Vercel に別プロジェクトとして（GitHub 連携で push→自動デプロイ）
- **リポジトリ**: この `realyou-lp/` ディレクトリを独立 git リポとして管理（real-you / 2026_team6 とは分離）

### ディレクトリ構成（想定）

```
realyou-lp/
├─ docs/spec.md          ← 本仕様書
├─ public/               ← 画像・ロゴ・システム構成図などの静的素材
├─ src/
│  ├─ pages/index.astro  ← トップ（1 ページ）
│  ├─ components/        ← セクションごとにコンポーネント分割
│  └─ styles/
├─ astro.config.mjs
└─ package.json
```

- セクションごとに Astro コンポーネントへ分割する（Hero.astro / Problem.astro / ... ）
- 素材は `KC3-2026/assets/` から `public/` へコピーして使う（real-you と同じ運用）

### 4-1. Hero
- プロダクト名：**Real You 〜本当の私じゃだめですか？〜**

### 4-2. 課題
- 性格診断のバイアス問題を 1〜2 文で。
- 「MBTI などの診断は“こうありたい自分”で答えてしまう。本当の自分は分からないまま」という温度感。

### 4-3. 何を測っているか
- 「**ゲーム中の“無意識の行動”を計測し、5 つの軸で本当の性格を診断する**」というレベルの説明に留める。
- **生データの取得方法・スコアリングの数理的な詳細は書かない**（「ふーん」となるため）。
- 5 つの診断軸を見せる：**慎重さ / 冷静さ / 論理性 / 協調性 / 積極性**
- 3 つのゲームを簡潔に紹介（各カード 1 枚程度）：
  - **Game 1：規約の罠** — 利用規約同意画面に見せかけた計測
  - **Game 2：荷物仕分け** — ベルトコンベアの仕分けゲームで判断プロセス・冷静さを計測
  - **Game 3：空気読み** — 職場チャットで同調するか意見を貫くかを計測

## 7. Astro 初学者メモ（こせ向け）

React/Next.js を触ったことがある前提での、Astro の最低限のつかみどころ。実装を読む/触る前に目を通す用。

### Astro とは
- **コンテンツ中心の静的サイト向けフレームワーク**。デフォルトでブラウザに JS をほぼ送らない＝表示が速い。今回のような紹介サイトと相性が良い。
- 今回は **SSG（静的生成）専用**で使う。ビルドすると素の HTML/CSS が `dist/` に吐かれ、Vercel はそれを配るだけ。サーバーは動かない。

### `.astro` ファイルの構造（一番大事）
```astro
---
// ① front matter：ここは「ビルド時」に1回だけ実行される JS/TS
// import やデータ取得、props の受け取りを書く。ブラウザには送られない
import Hero from '../components/Hero.astro';
const title = 'Real You';
---
<!-- ② テンプレート：HTML をそのまま書く。{title} で上の変数を埋め込める -->
<main>
  <h1>{title}</h1>
  <Hero />
</main>
<style>
  /* ③ ここに書いた CSS は、この .astro ファイルの中だけに自動で閉じる（scoped） */
  h1 { font-weight: 900; }
</style>
```
- React の JSX と似ているが **JSX ではない**。`className` ではなく **`class`** を使う（HTML そのまま）。
- 1 ファイル = 1 コンポーネント。`import` して `<Hero />` のように置くだけ。値は props（`<Hero title="..." />`）で渡す。

### ルーティング
- **ファイル＝URL**。`src/pages/index.astro` が `/`（トップ）になる。今回は 1 ページなので基本これだけ。

### 画像・静的ファイル
- `public/` に置いたものは、**そのままのパスで配信**される。`public/logo.png` → `<img src="/logo.png">`。

### クライアント JS は「必要な箇所だけ」
- Astro はデフォルトで JS を送らない。ボタンの動きなど JS が要る部分だけ `client:*` ディレクティブで明示する（例：`<Counter client:load />`）。**今回の紹介サイトはほぼ不要**な想定。

### よく使うコマンド
| コマンド | 役割 |
|---|---|
| `npm install` | 依存インストール |
| `npm run dev` | ローカル開発サーバー（`astro dev`）。保存で自動リロード |
| `npm run build` | 本番ビルド（`dist/` 生成） |
| `npm run preview` | ビルド結果をローカルで確認 |

### Next.js との主な違い（混乱しやすい点）
- `className` → **`class`**
- `useState` 等のクライアント状態は基本使わない（静的サイトなので）。動きが要る所だけ `client:*`。
- ルーティングは App Router のような規約ではなく、**素朴に `pages/` 配下のファイル＝パス**。
- スタイルは `<style>` がそのファイルに自動 scope されるので、CSS Modules 的なことを意識しなくてよい。
